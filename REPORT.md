## 2. 产品愿景与战略定位

### 2.1 愿景

> **让每一个国标麻将玩家，在做每一次决策时，都能基于事实而不是感觉。**

这不是一句口号——它定义了我们的产品边界和判断标准：
- **不是**自动打牌机器人（代替决策）
- **不是**算番器（只有胡牌结果）
- **是**决策副驾（分析所有选项，交给玩家判断）

### 2.2 使命

为进阶麻将玩家提供一个**可解释的、数学驱动的、隐私优先的**实时决策支持工具。

### 2.3 战略定位

当前市场存在一个明显的空白地带：

```
                   算番器
                （只回答"胡了多少分"）
                    ↑
         ┌──────────┼──────────┐
         │          │          │
       简单 ←────────────────→ 全面
         │          │          │
         └──────────┼──────────┘
                    ↓
                麻将 AI
              （黑盒，不可解释）
```

**空白地带 = 解释型决策支持**：
- 比算番器多一个维度：**胡牌前**就给出建议
- 比麻将 AI 多一个维度：**可解释**——每个建议附带数学依据
- 比两者都多一个维度：**隐私优先**——不上传任何牌局数据

### 2.4 核心原则

| 原则 | 解释 | 影响决策的例子 |
|---|---|---|
| **解释优先** | 不给结论，给依据 | 不弹窗说"打1万"，而展示"打1万→听牌，进张12张" |
| **透明度** | 所有可见信息都展示 | 剩余牌池、各对手舍牌、安全度一一列出 |
| **数学驱动** | 所有建议来自计算 | 不用经验规则，用 Monte Carlo 模拟 |
| **隐私优先** | 所有计算在本地完成 | 不上传手牌数据，不联网也可用 |

### 2.5 不做的事

| 不做 | 理由 |
|---|---|
| 自动打牌 / 作弊工具 | 违反产品价值观 |
| 实时牌局识别（OCR/摄像头）| 技术复杂、偏离核心 |
| 联网对战平台 | 非决策工具范畴 |
| 移动端原生 App | Web 已覆盖，原生维护成本过高 |
| 非国标规则支持（日本麻将等）| 番种系统完全不同 |

## 3. 市场洞察与用户需求

### 3.1 市场背景

国标麻将是目前规则最完整的麻将竞技形式——81种番种、8番起胡、88番封顶。但在实际对局中，存在严重的**信息不对称**：

- 高手靠经验直觉——但说不清为什么
- 中手靠死记硬背——记不全81种番种
- 新手靠运气——输都不知道怎么输的

### 3.2 目标用户金字塔

```
          ┌──────────┐
          │ 直播主播  │  ~5-10万  │  高价值用户
          │  5-10万   │          │  决策影响收入
         ┌┴──────────┴┐
         │  进阶玩家   │  ~100万  │  核心用户群
         │  ~100-200万 │          │  有提升需求
        ┌┴────────────┴┐
        │   麻将爱好者  │ ~500万   │  潜在用户
        │   周活3次+    │          │  可通过口碑转化
        └──────────────┘
```

**第一阶段目标层**：进阶玩家 ≈ 100-200万

### 3.3 用户痛点验证

基于调研（n≈50，线上+线下），我们对痛点做了优先级排序：

| 痛点 | 提及率 | 严重程度 | 产品对应功能 |
|---|---|---|---|
| 番种太多记不全 | 92% | ⭐⭐⭐⭐⭐ | 番种明细面板 |
| 听牌选错亏番数 | 85% | ⭐⭐⭐⭐⭐ | 听牌推荐+替换对比 |
| 不知道打哪张最危险 | 78% | ⭐⭐⭐⭐ | 防守安全度矩阵 |
| 算番慢，打完才知差点胡 | 70% | ⭐⭐⭐⭐ | 实时番数计算 |
| 吃碰杠犹豫不决 | 65% | ⭐⭐⭐ | 操作前后对比 |

**核心发现**：用户最深的痛不是"不知道规则"，而是"**选择困难**"——面对多选项时缺乏量化依据。

### 3.4 竞品格局

| 类型 | 代表 | 他们擅长 | 短板 |
|---|---|---|---|
| 算番器 App | 多款工具 | 胡牌后番数计算 | 胡牌前决策不支持 |
| 麻将 AI | Suphx（微软）| 胜率极高 | 黑盒，不可解释，不开源 |
| 游戏内辅助 | QQ麻将等 | 体验集成 | 绑定商业平台，规则限制 |
| 教学软件 | 多款 | 预设场景教学 | 不能用于真实对局 |

**本项目的差异化**：唯一一个同时满足"全决策链 + 可解释 + 离线隐私"的工具。

### 5.3 算法核心：向听数

向听数是整个决策树的基础——所有出牌建议、进张分析、听牌推荐都建立在向听数之上。

```
向听数 = min(标准型, 七对型, 十三幺型, 组合龙型)

标准型算法：
  1. 提取所有刻子/顺子（面子）
  2. 剩余牌贪心配对
  3. 计算缺失的面子数
  4. 补正雀头因素

时间复杂度：O(34 × k)，k < 10
```

**已验证正确性**：全部标准手牌类型（单面听/两面听/多面听/不靠/七对）已通过测试验证。

### 5.4 算法核心：蒙特卡洛模拟

MC 是本项目区别于算番器的关键差异——它不只是算番，而是评估**未来发展**。

```
每张候选出牌  →  模拟 N=500 局
     │
     1. 从剩余牌池随机摸 8 张
     2. 检查是否胡牌
     3. 能胡 → 计算番数
     │
     ▼
输出: 胜率 = 胡牌局数/N
      均番 = 总番数/胡牌局数
      EV   = 胜率 × 均番
```

**性能**：8 张牌 × 500 局/张，8 核并行 → < 3秒

**局限性**：模拟假设对手不出牌（仅摸牌），胜率被高估。但用于**排序**仍然有效。

#### 迭代 007：防守引擎升级（当前）

**现状**：防守基于全局舍牌统计，不区分对手
**目标**：安全度矩阵——每张牌对每家对手独立评分 0-100

**安全度评分逻辑**：

```
基线（基于牌类型+这张牌出过的张数）：
  字牌+0张=40   字牌+1张=70   字牌+2张=95
  幺九+0张=15   幺九+1张=50   幺九+2张=80
  中张+0张=5    中张+1张=35   中张+2张=75

调整（基于对手信息）：
  对手有同花色刻子 → -20
  对手有多个字牌刻子 → -30
  对手刚出过这张 → +20

综合：clamp(调整后, 0, 100)
```

**测试**：新增 15 项 → 总测试 212
