### 迭代 007: 防守引擎升级（安全度矩阵）

**目标**：从"按舍牌统计分危险/安全"升级为按对手建模的安全度矩阵

**改动清单**：

| 文件 | 改动 | 复杂度 |
|---|---|---|
| `decision/game_engine.py` | `analyze_defense` → 每个对手独立安全度 | ⭐⭐⭐ |
| `decision/game_engine.py` | 新增 `SafetyMatrix` 数据类型 | ⭐⭐ |
| `core/tile.py` | 新增 `classify_tile_type(code) → "字牌"|"幺九"|"中张"` | ⭐ |
| `tests/test_defense.py` | 新增 15-20 项防守测试 | ⭐⭐ |

**安全度模型**：

```
SafetyMatrix (34张 × 3对手):
  - 安全度: 0(极危) ~ 100(绝对安全), 分 5 级
    - 0-20: 高危（生牌且中张，有人大概率需要）
    - 21-40: 中危（生牌但字牌/幺九，或出过1张的中张）
    - 41-60: 警惕（出过1张的字牌/幺九）
    - 61-80: 安全（出过2+张）
    - 81-100: 绝对安全（河底/刚打过的同花色跟打）
  - 输入:
    - 每家舍牌记录（已知）
    - 每家副露（已知）
    - 剩余牌池（计算得出）
    - 上家/下家/对家位置差异
  - 规则:
    - 字牌: 出0=中危(40), 出1=安全(70), 出2+=绝对安全(95)
    - 幺九: 出0=高危(15), 出1=警惕(50), 出2+=安全(80)
    - 中张: 出0=极危(5), 出1=中危(35), 出2+=安全(75)
    - 中张且有人副露同花色刻子 → 降 20 分
    - 字牌且有人副露字一色方向 → 降 30 分
    - 有人出过同花色大面积中张 → 降 10 分
    - 跟打（有人刚出同张牌）→ +20 分
```

**API 改动**：

```
// /api/analyze 响应新增字段
defense: {
  safety_matrix: [
    { seat: 0, tile: "1wan", safety: 85, level: "安全", reason: "已出3张" },
    ...
  ],
  top_danger: [{ tile: "5tiao", safety: 5, level: "极危", reason: "生牌中张" }],
  top_safe: [{ tile: "东", safety: 95, level: "绝对安全", reason: "河底字牌" }],
  summary: "⚡ 危险: 5条·6条·8万  | 安全: 东·南·白"
}
```

**测试清单**（至少 15 项）：
1. 字牌出0张 → 安全度 40（中危）
2. 字牌出2张 → 安全度 95（绝对安全）
3. 中张出0张 → 安全度 5（极危）
4. 幺九出1张 → 安全度 50（警惕）
5. 中张有人副露同色刻子 → 降 20 分
6. 跟打（有人刚出同牌）→ +20 分
7. 手牌包含危险牌时标记高亮
8. 全安全度 34 张覆盖（不遗漏）
9. 排序：极危→高危→中危→警惕→安全→绝对安全
10. 3 家不同舍牌 → 安全度各不相同
11-15: 边界情况（剩余0张不计算、所有牌都出过等）

**验收标准**：
- 15 项测试通过
- API 返回 safety_matrix 字段
- 出牌建议中 danger_level 与 safety_matrix 一致
- 197 + 15 = **212 项测试**

### 迭代 012: 查叫预估引擎（对手听牌估算）

**目标**：基于各家舍牌+副露，估算对手是否听牌及可能胡的牌

**改动清单**：

| 文件 | 改动 | 复杂度 |
|---|---|---|
| `decision/opponent_model.py` | 新建：对手手牌概率模型 | ⭐⭐⭐⭐⭐ |
| `decision/game_engine.py` | `full_analysis` 集成查叫预估 | ⭐⭐ |
| `tests/test_opponent_model.py` | 新增 10 项测试 | ⭐⭐⭐ |

**对手建模方案**：

```
查叫预估（不依赖手牌推断，仅靠公开信息）:
  1. 输入: 各家舍牌 + 副露
  2. 输出: 每家"听牌概率" + "最可能胡的牌" top 5

  方法一（简单版，P0）:
  - 基于舍牌数量: 舍牌 N 张 → 至少摸过 N 次 → 越靠近后期越可能听牌
  - 听牌信号:
    - 连续出同花色中张 → 可能在做清一色
    - 出过很多字牌 → 已经清过牌，可能听牌
    - 突然换打同花色顺序 → 可能进张后听牌
    - 副露多(2+组)且手牌少 → 大概率听牌
  - 输出: {seat: 0-3, listen_probability: 0.0-1.0, likely_waits: [tile, ...]}

  方法二（蒙特卡洛版，P1+）:
  - 枚举对手可能的牌型（基于舍牌反向推断）
  - 蒙特卡洛模拟每种牌型 → 统计胡牌概率
  - 成本高，可能不做
```

**验收标准**：
- API 返回 `opponent_analysis` 字段
- 每家有听牌概率 + top-5 可能胡的牌
- 10 项测试通过
- 237 + 10 = **247 项测试**
